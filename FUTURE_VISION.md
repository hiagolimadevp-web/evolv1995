# EVOLV — Future Vision (10 Years)

This document is not a roadmap.
It is a set of principles for how EVOLV may grow into new surfaces, devices and
capabilities over the next decade — without losing its soul.

If a future decision contradicts this document, this document wins over the decision.

---

## The North Star

In 10 years, EVOLV should still feel like the same quiet companion it is today —
only present in more moments of life, and only where its presence adds calm rather than noise.

We will know we succeeded if users describe EVOLV the same way in 2035 as in 2025:
*"It's the one app that never makes me feel bad."*

---

## Guiding principles for every new surface

1. **Presence over notification.** A new surface must reduce the number of interruptions in a user's life, not add to them.
2. **Ambient over demanding.** The Companion should be felt more than seen.
3. **Optional by default.** Every new capability is off until the user chooses it.
4. **Portable identity.** The Companion is the same across every device — same tone, same restraint, same memory.
5. **Private by architecture.** New surfaces cannot exist if they require weakening our privacy stance.
6. **Consistency over novelty.** We adopt a new platform only when we can honor our philosophy on it — never because it's trendy.

---

## Apple Watch

A place for the smallest possible interactions: a glance, a breath, a single tap.
The Watch never nags. It never buzzes for a streak.
It is where the Companion whispers, not where it speaks.

## Wear OS

Same principles as the Watch. Feature parity in tone and restraint,
not necessarily in interface. If we cannot deliver calm on the platform, we don't ship on it.

## Smart Rings

Rings are quiet by nature — that matches EVOLV well.
Signals from a ring (sleep, recovery) are used to *soften* the day's suggestions,
never to grade the user.
A poor night is met with "take it easy today", never with a score.

## Cars

In a car, EVOLV becomes almost invisible.
Only voice, only the driver's choice, only content that helps them arrive calmer than they left.
Never metrics. Never data entry. Never anything that competes with the road.

## Voice

Voice is the most human surface, and therefore the one most easily broken by a fake tone.
The Companion's voice — whenever it exists — will be quiet, unhurried, and warm.
It never sells. It never upsells. It never "engages".

## Widgets

Widgets show one thing, gently. A gentle "you showed up yesterday" beats a dashboard.
If a widget cannot be glanced at in under a second, it is too much.

## Family Mode

Family Mode exists only if we can make it *support* without enabling control.
- No parent surveilling a teenager's weight.
- No spouse tracking another's calories.
- No leaderboards.
- Shared rituals, shared encouragement, shared silence — never shared metrics that can be weaponized.

If Family Mode ever risks becoming a tool of pressure, we remove it.

## AI Chat

As the Companion gains conversational ability, it must gain *more* restraint.
- It listens more than it responds.
- It never impersonates a therapist or doctor.
- It defers to professionals whenever the user's wellbeing may be at risk.
- It refuses to shame, judge, rank, or compare.
- It remembers the user's *why*, and returns to it gently in hard moments.
- When in doubt, it stays silent.

Silence is a feature.

## Health platforms (Apple Health, Health Connect, and beyond)

We read health data to make EVOLV kinder — never to make it louder.
- We only request the minimum data needed.
- We never sell, share, or monetize health data. Ever.
- We surface trends softly, and always with context.
- The user can revoke everything at any time, in one tap, and nothing is held hostage.

---

## What we will never do, no matter how much money is on the table

- Sell user data.
- Show ads inside EVOLV.
- Add social comparison features (leaderboards, public feeds, "friends passed you").
- Use fear, guilt, or streak-loss to drive retention.
- Add "AI features" that only exist to be marketed.
- Replace the human quality of the Companion with a generic chatbot personality.
- Ship a feature we would not want our own family to use exactly as it is.

---

## The 10-year test

Every year, we re-read this document and ask one question:

*"Does the EVOLV of today still feel like the EVOLV we promised?"*

If the answer is yes, we continue.
If the answer is no, we stop and repair — before we build anything new.
